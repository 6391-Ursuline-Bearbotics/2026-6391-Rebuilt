## Architecture

This is an FRC 2026 robot (Team 6391) built on the **AdvantageKit TalonFX Swerve Template** by Littleton Robotics. It uses WPILib's command-based framework with AdvantageKit for deterministic replay logging.

### AdvantageKit IO Pattern

Every subsystem follows this layered pattern:

1. **IO Interface** (`{Name}IO.java`) — defines an `@AutoLog` inner `Inputs` class and `default`-empty methods. The empty defaults allow `new IO() {}` for replay mode.
2. **Hardware Implementation** (`{Name}IOTalonFX.java`) — configures Phoenix 6 motors, reads status signals, uses `Debouncer(0.5, kFalling)` for connection monitoring.
3. **Sim Implementation** (`{Name}IOSim.java`) — uses `DCMotorSim` with WPILib physics simulation.
4. **Subsystem Class** — owns IO + `InputsAutoLogged`, calls `updateInputs()` then `Logger.processInputs()` in `periodic()`.

Mode-based instantiation happens in `RobotContainer` constructor via `switch (Constants.currentMode)`:
- `REAL` → hardware IO implementations
- `SIM` → sim IO implementations
- `REPLAY` → empty `new IO() {}` implementations

### Motor Configuration Convention (Phoenix 6)

```java
var config = new TalonFXConfiguration();
// ... set config fields ...
tryUntilOk(5, () -> talon.getConfigurator().apply(config, 0.25));
BaseStatusSignal.setUpdateFrequencyForAll(50.0, ...signals...);
ParentDevice.optimizeBusUtilizationForAll(talon);
```

Use `TalonFX(id, CANBus)` (not the deprecated `TalonFX(id, String)` constructor). The utility `PhoenixUtil.tryUntilOk()` is at `frc/robot/util/PhoenixUtil.java`.

### Alerts

WPILib `Alert` objects are created in constructors and toggled via `.set(boolean)` in `periodic()`. Used for motor disconnection (`kError`) and over-temperature (`kWarning`). Exclude SIM mode from disconnection alerts: `!inputs.connected && Constants.currentMode != Mode.SIM`.

### Constants

Constants are co-located with their subsystem (e.g., `IntakeConstants.java`, `VisionConstants.java`). The top-level `Constants.java` only contains the mode enum. `TunerConstants.java` in `generated/` is auto-generated by CTRE Tuner X.

## Subsystems

- **Drive** — Swerve with TalonFX modules, Pigeon2 gyro, high-frequency odometry thread, Choreo trajectory following, SysId
- **Vision** — Multi-camera (Limelight + QuestNav real, PhotonVision sim), AprilTag pose estimation
- **Intake** — Deploy motor (25:1, current-spike hard stop detection, state machine) + Roller motor (1:1, velocity control). Goal-based control: IDLE, INTAKE, EJECT, DEPLOYED_IDLE

## Controllers

- Port 0: Driver (`drv`) — drive modes, gyro reset
- Port 1: Operator (`op`) — intake controls (A=intake, B=idle, Y=deployed idle)

## CAN IDs

Swerve modules: FL(11,12,13) FR(21,22,23) BR(31,32,33) BL(41,42,43). Pigeon2: 1. Intake: Deploy=51, Roller=52 (placeholders in `IntakeConstants`). Default CAN bus (rio).

## Log Analysis Tools

Three Python scripts in `tools/` analyze AdvantageKit `.wpilog` files from competition.
**Requirement:** `pip install robotpy-wpiutil` (included with RobotPy). Run with `python -X utf8` on Windows to avoid encoding errors with degree symbols.

All three scripts share the same CLI pattern:
```
python -X utf8 tools/<script>.py  <log_directory>  [options]

Common options:
  -o report.txt      Save output to a file (also prints to console)
  --no-practice      Skip practice match logs (p## files)
  --match "Qual 5"   Analyze only one specific match
```

Log files are named `akit_<date>_<time>_<event>_<type><num>.wpilog` where type is `p`=practice, `q`=qual, `e`=elim.

---

### 1. `tools/log_analyzer.py` — General Drivetrain Health

**When to run:** After every competition day, or when asked about overall drive health, speed trends, battery, or loop timing.

```bash
python -X utf8 tools/log_analyzer.py "C:\path\to\logs" -o report.txt
python -X utf8 tools/log_analyzer.py "C:\path\to\logs" --no-practice
python -X utf8 tools/log_analyzer.py "C:\path\to\logs" -w 2.0   # wheel radius in inches
```

**Produces:**
- Swerve drive current (mean, p95, peak) per module per match — flags inter-module spread
- Drive + turn motor temperatures
- Wheel speed (mean & peak m/s) per module
- Acceleration / deceleration trends with bar chart — flags drops ≥20% vs previous match
- Battery voltage (mean, min, max) — flags brownout risk (<7V)
- RoboRIO loop time (mean, max, p95) — flags overruns >20ms
- Enabled duration per match — flags short/incomplete matches
- **Anomaly summary at the bottom** — all CRIT/WARN flags collected in one place

**Key thresholds to tune** (in `THRESHOLDS` dict at top of file):
- `current_mean_diff_warn/crit` — inter-module current spread (default 3A warn / 6A crit)
- `battery_mean_warn` — 11.0V is aggressive for competition; lower to 10.5V to reduce noise
- `short_match_thresh` — 60s default; adjust if pre-match testing inflates enabled time

### 2. `tools/slip_analyzer.py` — Swerve Wheel Slip / Free-Spin Detection

**When to run:** When a drive module is suspected of slipping, a wheel is damaged, or current draw is asymmetric between modules.

```bash
python -X utf8 tools/slip_analyzer.py "C:\path\to\logs"
python -X utf8 tools/slip_analyzer.py "C:\path\to\logs" --module 3   # focus on BL (default)
python -X utf8 tools/slip_analyzer.py "C:\path\to\logs" --module 2   # focus on BR
```

**Options:**
- `--module N` — which module to highlight in the CPR ratio summary (0=FL 1=FR 2=BR 3=BL)
- `--slip-vel` — rad/s threshold above which "slip events" are counted (default 40)
- `--slip-amp` — max amps at slip-vel to count as a slip event (default 15A)

**Produces:**
- **Current-Per-Rad/s (CPR) table** — lower = spinning fast for fewer amps = under-loaded wheel
- **CPR ratio (focus module vs avg of others)** — ratio < 0.94 flagged as low, < 0.88 as critical
- Pearson correlation between velocity and current per module
- Slip event rate (high vel + low current samples)
- **Velocity signature table** — median current at each speed bin (0-80 rad/s) for all 4 modules; a module consistently lower in the 10-40 rad/s range is under-loaded against carpet

**How to interpret CPR ratio progression:**
- Ratio stable near 1.0 across matches = healthy
- Ratio starting at 1.0 and dropping over matches = progressive mechanical wear
- Single-match spike in p95 current = possible debris/impact event

**STL competition findings (for reference):**
- BL (module 3) started healthy in Practice 4/7 (ratio ~1.03) and degraded to 0.815 by Elim 6
- Signature showed BL drawing 10-15% less current at 10-40 rad/s vs other modules
- Conclusion: BL bevel gear slip or wheel surface wear that developed during the event

### 3. `tools/mechanism_analyzer.py` — All Non-Drive Subsystem Motors

**When to run:** When asked about shooter, indexer, intake, or overall robot power. Also useful after matches where game piece handling was poor or a motor warning fired.

```bash
python -X utf8 tools/mechanism_analyzer.py "C:\path\to\logs" -o mechanisms.txt
python -X utf8 tools/mechanism_analyzer.py "C:\path\to\logs" --no-practice
```

**Produces:**
- **Shooter L/R** — mean/p95/peak current, CPR balance (which side is free-spinning), flywheel velocity imbalance %, temperatures, estimated shot event count
- **Indexer Belt** — current stats + jam event count (stall = high current + low velocity)
- **Indexer Kicker** — current trends
- **Indexer Spinners L/R** — current balance between left and right
- **Intake Deploy** — current + hard-stop spike count (stator >30A = arm hitting physical stop)
- **Intake Roller** — current, jam events, and **motor disconnection count** (falling edge on Connected during enabled)
- **PDH total current and power** (enabled)
- **Active alerts** — all WPILib Alert messages that fired during enabled periods, per match

**Key thresholds** (in constants at top of file):
- `BELT_JAM_AMP/VEL`, `ROLLER_JAM_AMP/VEL` — stall detection thresholds
- `DEPLOY_SPIKE_AMP` — hard stop current threshold (30A default)
- `TEMP_WARN` — motor temperature warning (55°C default)

**STL competition findings (for reference):**
- Intake roller disconnected 7× in Qual 5 and 3× in Qual 21 — highest priority for championship
- Intake roller ran 55-59°C peak — overworked; check roller surface and current limits
- Gyro disconnected during Practice 7 and Elim 6 — check Pigeon2 mounting/CAN under defense
- Shooter Right wheel consistently lower CPR — possible worn flywheel surface
- Indexer spinner Left draws ~3.5A more than Right in every match — mechanical asymmetry
- Indexer belt jams escalated in Elim 12 — check belt tension and game piece path
